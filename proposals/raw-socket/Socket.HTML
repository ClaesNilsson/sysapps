Interface Navigator

The RawSocket interface is exposed on the Navigator object.

readonly attribute RawSocket rawsocket
The object that exposes the rawsocket functionality.

RawSocket Interface

The RawSocket interface exposes a method for creating a socket.

Socket create()

Creates a socket of the specified type. 

            var socket = navigator.rawsocket.create('udp', {});

Throws DOMException INVALID_ACCESS_ERR if any input parameter contains an invalid value or if the input parameter is not compatible with the expected type for that parameter.

SocketType type
The type of socket to create. Must be tcp or udp.
optional CreateOptions options
The socket options.

This use of a Socket object, for which a set of methods is defined, is different from the Chrome Scoket API, which uses a "socket id" as input parmeter in all methods. An alternative would be to define a Socket interface constructor so that a Socket object for example is created with mySocket = new Socket (’tcp’);.

void bind()

Binds the local address for a UDP socket. Can not be used for TCP sockets.

Throws DOMException INVALID_ACCESS_ERR if any input parameter is not compatible with the expected type for that parameter.

BindCallback callback
Called when the bind attempt is complete.
optional DOMString address
The address of the local machine. TBD what applies when this par is not present (similar to '0.0.0.0' for Chrome Socket API)
optional unsigned short port
The port of the local machine. TBD what applies when this par is not present (similar to 0 for Chrome Socket API)
void connect()

Connects the socket to the remote machine (for a tcp socket). For a udp socket, this sets the default address that is used for read() and write() calls.

Throws DOMException INVALID_ACCESS_ERR if any input parameter is not compatible with the expected type for that parameter.

ConnectCallback callback
Called when the connection attempt is complete.
DOMString address
The hostname or IP address of the remote machine.
unsigned short port
The port of the remote machine.
void destroy()

Destroys the socket. Each socket created should be destroyed after use.

void disconnect()

Disconnects the socket. For UDP sockets, disconnect is a non-operation but is safe to call.

Is this method needed? Isn't it enough with destroy()?

void read()

Reads data from the given connected socket.

Throws DOMException INVALID_ACCESS_ERR if any input parameter is not compatible with the expected type for that parameter.

ReadCallback callback
Called when the read operation is complete.
optional long bufferSize
The max number of bytes to receive before callback.
void recvFrom()

Receives data from the given UDP socket.

Throws DOMException INVALID_ACCESS_ERR if any input parameter is not compatible with the expected type for that parameter.

RecvFromCallback callback
Called when the receive operation is complete.
optional long bufferSize
The max number of bytes to receive before callback.
void sendTo()

Sends data on the given UDP socket to the given address and port.

Throws DOMException INVALID_ACCESS_ERR if any input parameter is not compatible with the expected type for that parameter.

SendToCallback callback
Called when the send operation completes without blocking or an error occurs.
ArrayBuffer data
The data to write.
DOMString address
The address of the remote machine.
unsigned short port
The port of the remote machine.
void setKeepAlive()

Enables or disables the keep-alive functionality for a TCP connection.

Throws DOMException INVALID_ACCESS_ERR if any input parameter is not compatible with the expected type for that parameter.

SetKeepAliveCallback callback
Called when the setKeepAlive attempt is complete.
boolean enable
If true, enable keep-alive functionality.
optional long delay
The delay seconds between the last data packet received and the first keepalive probe. Default is 0..
void setNoDelay()

Sets or clears TCP_NODELAY for a TCP connection. Nagle's algorithm will be disabled when TCP_NODELAY is set.

Throws DOMException INVALID_ACCESS_ERR if any input parameter is not compatible with the expected type for that parameter.

SetNoDelayCallback callback
Called when the setNoDelay attempt is complete.
boolean enable
If true, disables Nagle's algorithm.
void write()

Writes data on the given connected socket.

Throws DOMException INVALID_ACCESS_ERR if any input parameter is not compatible with the expected type for that parameter.

WriteCallback callback
Called when the send operation completes without blocking or an error occurs.
ArrayBuffer data
The data to write.

So far support for server sockets through the methods listen() and accept() has been omitted. Not sure on use cases for server sockets.
